Refactor Suggestions

Timed recommendations — not blockers for the current build phase

When building import commands

Register models in Django Admin

Create core/admin.py and register all seven models. Gives you a free data browser to verify imports during development.

from django.contrib import admin
from .models import Album, Artist, Track, Run, RunPoint, PlayEvent, RunTrackMatch

admin.site.register(Album)
admin.site.register(Artist)
admin.site.register(Track)
admin.site.register(Run)
admin.site.register(RunPoint)
admin.site.register(PlayEvent)
admin.site.register(RunTrackMatch)

Use bulk_create for high-volume imports

RunPoint will import ~300K rows per user. Individual save() calls will be slow.

RunPoint.objects.bulk_create(points, ignore_conflicts=True)

When building the frontend

Move brand assets to static/

assets/wordmark.png should move to static/images/wordmark.png so Whitenoise can serve it in production.

Consider a map library

For an interactive route map, a JS library will be needed. Two options:

  • Leaflet.js — lightweight, easy to include as a static asset, good for basic colored polylines
  • MapLibre GL — WebGL-based, better for smooth gradient coloring along a route

Both can be included without a JS build pipeline.

When going to beta

Add password validators

# soundtraek/settings.py
AUTH_PASSWORD_VALIDATORS = [
    {'NAME': 'django.contrib.auth.password_validation.UserAttributeSimilarityValidator'},
    {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator'},
    {'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator'},
    {'NAME': 'django.contrib.auth.password_validation.NumericPasswordValidator'},
]

Add DATABASE_URL support for Supabase

Install dj-database-url and update settings.py to parse the connection string from the environment.

Consider RLS in Supabase

Row Level Security policies on Run and PlayEvent add a safety layer beyond Django's queryset filtering. Exempt Album, Artist, and Track (global shared data).
